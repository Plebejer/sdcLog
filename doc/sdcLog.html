Introduction to sdcLog

This vignette introduces the sdcLog package and its main functions, illustrated with various examples. The sdcLog package provides tools which simplify statistical disclosure control in the context of research data centres (RDSC).

The package includes three main functions:

  • sdc_descriptives(): This function is used for statistical disclosure control for descriptive statistics. It checks the data used for the calculation of descriptive statistics for compliance with the rules and regulations - set by the Research Data Centre. The calculation of simple extreme values (as conclusions can be drawn about individual statistical units) is not allowed according to the rules of the Research Data Center, so sdc_descriptives() cannot be used to check them. Extreme values can only be used if they were calculated with sdc_extreme().

  • sdc_extreme(): This function is used for the automatic calculation (if possible) of extreme values according to the rules of the research data centre. It uses the available data and calculates the values for desired variables and groupings in compliance with the rules. The values are calculated as averages of a sufficient number of observations. This helps the researcher to easily follow the rules and simplifies the control.

  • sdc_model(): This function is used for statistical disclosure control for various types of models. It checks the calculated model and the underlying data for compliance with the rules and regulations of the Research Data Center.

sdc_descriptives()

This function performs statistical disclosure control according to two main criteria. It checks for a sufficient number of different statistical entities (IDs) - listed here as distinct_ids. On the other hand, it checks for dominance, so that two statistical entities (IDs) do not account for more than 85 percent of the observed values. Following some examples and explanations for using sdc_descriptives().

sdc_extreme()

This function automatically calculates extreme values that comply with the rules of the Research Data Center. Again, it mainly checks the criteria of distinct_ids and dominance. The values are calculated as averages of a sufficient number of observations. It is based on an iterative procedure that aggregates data until there are enough distinct_ids to calculate the extreme values and no problems with dominance occur. The function always starts the iteration process with the lowest possible number of observations for each extreme value (here 5, since at least 5 distinct statistical units must be included in the calculation according to the rules of the research data centre). Furthermore, the function checks that the subset of data for minimum and maximum do not overlap.

If there are no problems with the calculation, the function returns a list with the extreme values - maximum and minimum are always output, no value of the two alone can be calculated separately. If it is not possible to calculate extreme values under these criteria, a corresponding message is printed and the list is returned with NA values. This is because the dominance criterion would be violated if only 5 observations were considered for the maximum. Thus 7 observations are automatically taken into account. If the researcher wishes to introduce an equal number of observations per extreme value, it is possible to specify this in the function using the arguments n_min and n_max. sdc_model()

This function checks if your model complies to RDSC rules. The criteria distinct_ids and dominance are also checked here. In addition, when using dummy variables, it is checked whether there are enough different observations (according to the rules of the RDSC) for each attribute/value. The function can be used to check a broad range of models like lm, glm and various others (anything which can be handled by [broom::augment()]).

Data & models

To introduce the function sdc_model(), a dataset with different variables is used, which include dummy-variables.

We have 80 observations of 10 different IDs for the variables ‘y’, ‘x_1’, ‘x_2’, ‘x_3’, ‘x_4’ and additional information on sector, year and country (dummy variables). A summary of the data set is given below.

Applying sdc_model()

The previously specified models are now checked for compliance with the rules of the RDSC. It is checked, if there are enough distinct_ids in the whole model and each model variable is checked if there are problems with dominance. Also every dummy variable is checked for compliance with the rules.

A pre-selection of problematic and unproblematic models has been made to better explain the differences. To check for compliance, the model object, the data used and the ID variable must be specified in sdc_model().
