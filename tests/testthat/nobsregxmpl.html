Create artificial data

Create artificial data for eleven IDs (1-11), ten years (2001-2010), three countries (17, 39, 400), and continuous variables x, y, and z.

. clear

. qui set obs 301

. gen id   = int((_n-1)/30)+1

. gen hv   = _n-30*(id-1)

. gen year = 2000+mod(hv,10)+10*(mod(hv,10)==0)    

. gen land = int((2000-year+hv)/10)+1 

. qui replace land =  17 if land==1

. qui replace land =  39 if land==2

. qui replace land = 400 if land==3    

. drop hv

. set seed 4869382

. gen x = runiform()

. gen y = runiform()

. gen z = runiform()

Create an id, that has to be ignored (since x is missing, see Create missings)

. qui replace id = 11 if _n==301

It is not permitted to publish information on specific firms. Create a dummy variable if id==1

. gen id1 = id==1

The dummy variable dum_5_57 (id/year/land: 1/2005/17, 6/2007/17) is 1 for only two distinct ids. Therefore, it must not be published. In contrast, the dummy variable dum_5_57_178 (id/year/land: 1/2005/17, 6/2007/17, 8/2008/39) is 1 for three distinct ids and can be published.

. qui gen dum_5_57     = _n==5|_n==57  

. qui gen dum_5_57_178 = _n==5|_n==57|_n==178

Create missings Often, yearly dummies are used as catch all dummies. However, a dummy for the year 2002 would identify the ids 1 and 2. Therefore, the coefficient for the 2002-dummy causes a disclosure problem.

. qui replace x = . if year==2002 & id > 2        

. qui replace x = . if id==11

Change the variable z so that it has for one id only values different from zero. Then this id is identified by z and no regression coeffcient for z must be published. nobsreg creates a dichotomous auxiliary variable that is 0 if z = 0 and 1 if z <> 0. If z = 1 for only one or two ids a disclosure problem is reported.

. qui replace z = 0 if id!=1

Later on there is an example using two different identifiers. This occurs e.g. if the data includes domestic firms as well as their affiliates abroad. Then confidentiality has to be assured for both, parent institutes and subsidiaries.

. gen idtest = id         

. qui replace idtest = 5 if year==2010

Applying nobsreg

How to use nobsreg with and without global idlist

All data disclosure control routines require a variable denoting the identifier of the entity to be protected. nobsreg allows two alternatives. The first one consists in writing this variable after nobsreg. The other is specifying a global macro idlist. This idlist may consist of more than one identifier because it is not always sufficient to investigate confidentiality for one identifier only. E. g. a sample may consist of two MFIs with each having three subsidiaries abroad. Then confidentiality has to be assured for both, parent institutes and subsidiaries. nobsreg loops through the idlist.
